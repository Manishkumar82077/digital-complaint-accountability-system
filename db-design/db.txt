================================================================
RELATIONSHIPS BETWEEN TABLES
================================================================

USERS
├── Has one: Admin profile (optional)
├── Has one: Professional profile (optional)
├── Has many: Complaints (as raiser)
├── Has one: Department
├── Has one: Manager (self-reference to Users)

ADMINS
├── Belongs to: User (one-to-one)
├── Has many: Complaint overrides
├── Has many: Admin actions in Audit Logs
├── Reports to: Admin (self-reference)
├── Manages: Departments (optional scope)

PROFESSIONALS
├── Belongs to: User (one-to-one)
├── Has one: Department
├── Has many: Complaints (as assigned professional)
└── Has many: Complaints (as resolver)

DEPARTMENTS
├── Has many: Users
├── Has many: Professionals
├── Has many: Complaints
├── Has one: Head Professional
├── Has one: Admin Owner
└── Has one: Parent Department (self-reference)

COMPLAINTS
├── Belongs to: User (raiser)
├── Belongs to: Professional (raiser, assigned, resolver)
├── Belongs to: Admin (override, assignment, escalation, closure)
├── Belongs to: Department
├── Has many: Evidence
├── Has many: Audit Logs
└── Has one: Parent Complaint (self-reference)

EVIDENCE
├── Belongs to: Complaint
├── Uploaded by: User, Professional, or Admin
└── Access overridden by: Admin (optional)

AUDIT_LOGS
├── Actor: User, Professional, or Admin
├── Approved by: Admin (for critical actions)
└── References: Any entity (polymorphic relationship)

ADMIN TYPES & LEVELS
================================================================

SUPER_ADMIN:
- Full unrestricted access
- Can manage other admins
- Can modify system configuration
- No department/location restrictions

SYSTEM_ADMIN:
- Full access to complaints and users
- Can manage professionals and departments
- Cannot modify other admins
- Global scope

DEPARTMENT_ADMIN:
- Limited to specific departments
- Can manage users in their departments
- Can handle complaints in their scope
- Cannot override super admin decisions
